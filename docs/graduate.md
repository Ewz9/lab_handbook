# 畢業規範

本實驗室畢業流程除須符合系所與學校規定外，亦須完成實驗室之研究與交接要求。

---

## 畢業標準檢核表

<div style="background-color:#fff8e1; padding:14px; border-radius:8px; border-left:5px solid #f0ad4e; margin:16px 0;">

<strong>重要提醒</strong><br><br>
指導教授將依此清單核對進度。若未完成實驗室規範，指導教授有權暫緩相關簽核流程。

</div>

### 1. 基礎規定與學術發表

- [ ] **滿足系所規定**：修課學分、成績達標及符合修業年限。
- [ ] **完成研究發表**：至少參與並協助完成一篇期刊或研討會論文，且須具備實質貢獻。
- [ ] **通過論文口試**：依口試委員意見完成論文修改與最終定稿。

### 2. 實驗室研究交接（核心要求）

為了確保研究成果的可重現性，必須完成以下事項：

- [ ] **程式碼整理**：所有程式碼須上傳至指定 GitHub Repository。
- [ ] **文件化（README）**：必須包含環境配置、執行指令、資料集來源說明。
- [ ] **資料歸檔**：論文原始檔案、Final 投影片及相關實驗數據。
- [ ] **操作示範**：親自帶領接手成員運行一遍程式，確保對方能獨立操作。

---

## 交接完成判定標準

<div style="background-color:#f3faf5; padding:14px; border-radius:8px; border-left:5px solid #4caf50; margin:16px 0;">

<strong>成功交接的定義</strong><br><br>

未達以下標準者，視為未完成交接：

<ol>
  <li><strong>可執行性</strong>：Repository 在新環境下可成功 <code>git clone</code> 並順利執行。</li>
  <li><strong>可復現性</strong>：README 文件讓接手者能獨立復現，不需口頭詢問。</li>
  <li><strong>完整性</strong>：論文與發表相關的原始圖檔、投影片皆已分類整理交予教授。</li>
</ol>

</div>

## 補充說明

- **循序漸進**：畢業流程須依序完成，請預留至少 1–2 週進行資料整理。
- **研究誠信**：所有數據與資料須確保可重現性。
- **詳細操作**：具體步驟請參閱 [行政事務與報帳流程 > 實驗室離開前業務交接](https://ewz9.github.io/lab_handbook/workflow/)。
